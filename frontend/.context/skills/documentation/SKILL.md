# Documentation Skill (Frontend)

Maintaining the frontend knowledge base for Clínica Go.

## Standards

- **Core Docs**: Reside in `frontend/.context/docs/`.
- **Component Docs**: Living styleguide at the `/styleguide` route.
- **Code Comments**: Use JSDoc for complex logic in services or hooks.

## Key Documentation Touchpoints

### 1. Styleguide Integration
Every new UI component in `src/components/ui/` MUST have a corresponding showcase page in `src/app/styleguide/components/`.
- **Purpose**: Visual documentation and manual QA.

### 2. .context/docs Updates
- **architecture.md**: Update when adding new global providers or major architectural shifts (e.g., adding a new state management lib).
- **development-workflow.md**: Keep updated with new npm scripts or CI/CD changes.
- **glossary.md**: Add frontend-specific UI terms or domain concepts (e.g., "Kanban view", "Metric Gauges").

### 3. README synchronization
Ensure the `docs/README.md` index remains accurate when new guide files are added.

## Conventions
- **Screenshots**: When possible, use images generated by `generate_image` or real screenshots to document UI expectations.
- **Diagrams**: Use Mermaid for component tree or state flow diagrams.
